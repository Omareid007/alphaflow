## ğŸ’¡ Best Practices Established

### Specification Writing

1. **Use SHALL/MUST** for normative requirements
2. **Include scenarios** for every requirement (minimum 1)
3. **Follow WHEN/THEN/AND** format consistently
4. **Reference implementation** files (e.g., `server/routes/auth.ts:42`)
5. **Validate continuously** with `openspec validate --strict`

### Change Management

1. **Create proposal first** - Don't implement without approval
2. **Use descriptive IDs** - `add-`, `update-`, `remove-` prefixes
3. **Document impact** - List affected specs and code
4. **Break down tasks** - Implementation checklist in tasks.md
5. **Archive after deployment** - Update specs when complete

### Documentation Maintenance

1. **Keep specs synchronized** - Update with code changes
2. **Use delta operations** - ADDED, MODIFIED, REMOVED, RENAMED
3. **Validate before committing** - Ensure 100% pass rate
4. **Archive completed changes** - Keep change history clean
5. **Reference across systems** - Link OpenSpec â†” OpenAPI â†” Code
